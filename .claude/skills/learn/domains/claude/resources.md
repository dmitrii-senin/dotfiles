# Resources: claude — vetted, citeable sources

Cite these over parametric memory, especially for limits, flag names, pricing, and
feature behavior (these change often). Flag anything unattributable as `⚠ unsourced`.

## Primary / authoritative
- **Claude Code docs** — https://code.claude.com/docs/en/overview (all pages: skills,
  hooks, MCP, plan mode, settings, permissions, subagents/workflows, statusline).
- **Anthropic docs** — https://docs.anthropic.com/en/docs (models, tokens, context,
  **prompt caching**, pricing, tool use, batch, effort/thinking).
- **claude-code repo / changelog** — https://github.com/anthropics/claude-code (new
  features, flags, release notes).

## Secondary
- **agentskills.io** — community skill patterns and best practices.
- **`references/best-practices.md`** — the local audit checklist / skill-authoring rubric.
- **The user's own setup** — `~/x/dotfiles/.claude/` (settings, skills, statusline) and
  project `CLAUDE.md` files: the source of truth for anchored, non-generic examples.

> When a precise figure or flag matters and isn't in `knowledge/`, attribute it to the
> docs above. If you can't, say so and point to the page to check.
