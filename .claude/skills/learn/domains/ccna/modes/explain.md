# `explain` mode (ccna) — direct explanation

Direct, structured explanation of a CCNA concept. Invoked as `/learn ccna explain <concept>`
(alias `/ccna explain <concept>`). Pure Cisco/networking framing — no cross-domain analogies
(Linux, other certs) unless the user explicitly asks.

## Structure
1. **Essence** — one line: what this thing actually is.
2. **How it works** — mechanism, ~5–15 lines.
3. **Cisco specifics** — IOS commands, syntax quirks, default values (cite the OCG / Cisco docs
   for exact numbers; flag anything unsourced).
4. **Common exam trap** — the one specific thing the exam asks about.
5. **Dig deeper** — point to the relevant `knowledge/N-*.md`.

Read the matching `knowledge/` file first (per `domain.md`'s area map). Stay on the blueprint.
